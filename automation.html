01 // The Brief

TheProblem

Manual → Automated
Front.Desk.Sys

Before

A service business in Dubai fields the same questions all day — pricing, availability, booking — across WhatsApp, Instagram, Facebook, email and the website. Every message answered by hand. Slow replies lose leads; late nights lose sleep.

After

Every channel routes into one shared AI engine that knows the business — services, prices, service areas — answers instantly in the customer's language, captures the lead, books the job on the calendar, chases the review, and reports the numbers. Staff only step in when it matters.

03 // Architecture.Notes

Built toNot Break

Hub.And.Spoke
Asia/Dubai

Channels are thin. Intelligence lives in one sub-workflow, errors flow to one handler, and every constant is marked for handoff — a system a client can own.

12Workflows.Deployed
06Intake.Channels
10+Service.Integrations
01Shared.AI.Engine
Phase.01

Design

Mapped every customer touchpoint into a hub-and-spoke system: six thin intake spokes, one AI hub, one error drain. Postgres schema for leads, bookings and conversation memory.

Phase.02

Build

Twelve workflows built and wired in n8n — Claude API for the brain, Graph API for Meta channels, Gmail, Google Calendar, Supabase, Notion. Setup guides written for the credential handoff.

Phase.03

Deploy

Suite deployed to n8n Cloud with sub-workflows linked by ID, EDIT-marked constants for the client's keys, and Docker + self-host docs so the system outlives any one platform.
